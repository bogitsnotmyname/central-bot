# Как запустить бота без вашего компьютера (Railway)

Бот будет работать 24/7 на сервере Railway — ваш компьютер можно выключать,
бот продолжит отвечать в Telegram.

## Что уже готово в этой папке

```
bot.py            — код бота (без изменений)
requirements.txt  — зависимости
Procfile          — команда запуска для Railway
.env.example      — список нужных переменных (без значений — впишете сами)
```

`bookings.db` **не включён специально** — Railway создаст пустой файл сам
при первом запуске бота. Если у вас уже есть записи в текущем `bookings.db`
и вы хотите их сохранить — шаг про это есть ниже, в разделе "Перенос
существующих записей".

## Шаг 1 — Установите Railway CLI

Это делается один раз, на вашем компьютере (сам бот после этого будет жить
на сервере Railway, не у вас).

**Mac/Linux:**
```bash
curl -fsSL https://railway.app/install.sh | sh
```

**Windows (PowerShell):**
```powershell
iwr https://railway.app/install.ps1 -useb | iex
```

Если это не сработает — на railway.app есть кнопка "Deploy" и веб-интерфейс
без CLI, но через CLI проще всего в вашем случае.

## Шаг 2 — Зарегистрируйтесь и войдите

1. Зайдите на [railway.app](https://railway.app), зарегистрируйтесь (можно
   через GitHub-аккаунт или e-mail)
2. В терминале, в этой папке с ботом, выполните:
   ```bash
   railway login
   ```
   Откроется браузер — подтвердите вход.

## Шаг 3 — Создайте проект и загрузите бота

Находясь в папке с `bot.py`:

```bash
railway init
```
Дайте проекту имя, например `central-alley-bot`.

```bash
railway up
```
Это загрузит файлы на Railway и запустит сборку. Подождите, пока
закончится (обычно 1–2 минуты).

## Шаг 4 — Впишите переменные окружения

Так как я не кладу ваш реальный токен в файлы, сделайте это на сайте Railway:

1. Откройте [railway.app/dashboard](https://railway.app/dashboard), зайдите
   в свой проект
2. Откройте вкладку **Variables**
3. Добавьте по одной все переменные из вашего `.env` (у вас уже есть готовые
   значения — просто скопируйте их оттуда):
   - `BOT_TOKEN`
   - `ADMIN_ID`
   - `TIMEZONE`
   - `WORK_START`
   - `WORK_END`
   - `SESSION_MINUTES`
   - `BOOKING_DAYS_AHEAD`
   - `SCHEDULE_DAYS_AHEAD`
   - `REMINDER_MINUTES_BEFORE`
   - `DB_PATH` — поставьте просто `bookings.db`
4. Railway сам перезапустит бота после сохранения переменных

## Шаг 5 — Проверьте, что команда запуска верная

В настройках проекта (Settings → Deploy) убедитесь, что **Start Command**
стоит:
```
python bot.py
```
(Обычно Railway подхватывает это автоматически из `Procfile`, но лучше
проверить.)

## Шаг 6 — Проверка

Откройте Telegram, напишите боту `/start`. Если ответил — всё работает, и
уже не важно, включён ваш компьютер или нет.

Проверить логи бота (например, если что-то не отвечает):
```bash
railway logs
```

## Перенос существующих записей (если нужно)

Если в вашем текущем `bookings.db` уже есть реальные бронирования и вы
хотите, чтобы они не потерялись:

1. В Railway откройте вкладку **Variables**, посмотрите, есть ли volume
   (постоянный диск) — если нет, добавьте: Settings → Volumes → New Volume,
   примонтируйте на `/data`, и поставьте `DB_PATH=/data/bookings.db`
2. Через `railway run` или встроенный файловый доступ загрузите ваш текущий
   `bookings.db` в этот volume — если понадобится помощь с этим шагом,
   напишите, разберём отдельно, это не сложно, но лучше сделать аккуратно.

**Важно про volume:** без него Railway может стирать `bookings.db` при
каждом новом деплое (это упомянуто и в вашем README). Если планируете
дальше обновлять код бота, настройте volume сразу — иначе рискуете
потерять записи клиентов при следующем обновлении.

## Если что-то пошло не так

Частые причины:
- **Бот не отвечает** — проверьте `railway logs`, скорее всего забыта одна
  из переменных (`BOT_TOKEN` или `ADMIN_ID`)
- **Бот всё время перезапускается** — обычно тоже нехватка переменной или
  опечатка в значении
- **Записи пропадают после обновления кода** — нужен Volume (см. выше)

Если пришлёте текст ошибки из `railway logs` — разберём вместе.
